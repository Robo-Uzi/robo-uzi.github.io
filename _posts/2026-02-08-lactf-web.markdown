Contents of `flag.js`:
```shell
const http = require("http");  
  
const FLAG = process.env.FLAG || "lactf{fake_flag}";  
const PORT = 8081;  
  
const server = http.createServer((req, res) => {  
 if (req.method === "GET" && req.url === "/flag") {  
   res.writeHead(200, { "Content-Type": "text/html" });  
   res.end(`<div><strong>FLAG:</strong> ${FLAG}</div>`);  
   return;  
 }  
 res.writeHead(404, { "Content-Type": "text/plain" });  
 res.end("Not found");

 console.log(`Flag service running on port ${PORT}`);

```

The input from the user is not sanitized! It is then directly embedded into the HTML. The HTML is rendered server side using Puppeteers `page.setContent()` which causes chromium (the invoice generator uses Puppeteer, which launches a bundled headless chromium browser I come to find out) to fetch external resources during rendering. Since the invoice generator container shares a docker network with the internal flag service, a injected HTML element can trigger server side requests to the internal endpoint. The response is rendered into the generated PDF. This results in a server side HTML injection leading to SSRF.
